PID Tuning Tool — 사용 공식 레퍼런스
본 문서는 PID Tuning Tool에서 모델 식별, 파라미터 추천, 시뮬레이션, 루프 진단에 사용되는
핵심 수식과 그 이론적 배경, 적용 조건을 정리한 레퍼런스입니다.
ℹ️  표기 규칙 — Kp: 프로세스 이득  |  τ: 시정수 (min)  |  θ: 데드타임 (min)  |  λ: IMC/Lambda 튜닝 파라미터 (min)  |  r = θ/τ: 정규화 데드타임 비율
📐
FOPDT 프로세스 모델
Process Identification
전달함수 (Transfer Function) S-Domain
G(s) = Kp · e−θs / (τs + 1)
Kp
프로세스 이득 (Process Gain)
ΔPV(steady) / ΔMV
τ (tau)
시정수 (Time Constant)
스텝 응답에서 최종값의 63.2% 도달 시간
θ (theta)
데드타임 (Dead Time / Delay)
MV 변화 후 PV가 반응하기 시작할 때까지 지연시간
e−θs
수송 지연 항 (Transport Delay)
순수 시간 지연을 라플라스 도메인에서 표현
🔬 식별 방법: 스텝 응답(PRBS 또는 수동 MV 스텝) 데이터에 비선형 최소제곱법을 적용하여 Kp, τ, θ를 추정. R² ≥ 0.85 이상이면 신뢰성 있는 모델로 판단.
Kp 추정 — 이득 계산
Kp = (PVfinal − PVinitial) / (MVfinal − MVinitial) ※ 단위: [%PV / %MV] 또는 [EU / %]
τ, θ 추정 — 2점법 (Two-Point Method)
t28 = PV가 총 변화량의 28.3%에 도달한 시각 t63 = PV가 총 변화량의 63.2%에 도달한 시각 τ = 1.5 · (t63 − t28) θ = t63 − τ
⚡ 빠른 현장 추정용. 비선형 최소제곱법보다 정밀도가 낮지만, 노이즈가 많은 데이터에서 강건한 초기값 제공.
정규화 데드타임 비율 (r)
r = θ / τ r < 0.3 : 제어 양호 (fast loop) | 0.3 ≤ r < 1.0 : 보통 | r ≥ 1.0 : 어려운 루프 (Dead-time dominant)

PID 제어기 표준형
ISA Standard
병렬형 PID (Parallel Form) Ideal
u(t) = Kc · [ e(t) + (1/Ti)∫e(t)dt + Td·de(t)/dt ] e(t) = SP(t) − PV(t)   (오차 신호)
Kc
비례 이득 (Controller Gain)
단위: 무차원 또는 %/%
Ti
적분 시간 (Integral Time)
단위: min (또는 sec)
Td
미분 시간 (Derivative Time)
단위: min (또는 sec)
e(t)
오차 = SP − PV
실시간 편차 신호
Laplace 도메인 PID 전달함수
C(s) = Kc · (1 + 1/(Ti·s) + Td·s)
직렬형 PID → 병렬형 변환 (Series ↔ Parallel)
// 일부 DCS는 직렬형(실제형)을 사용함. 아래는 등가 변환식: Kcparallel = Kcseries · (1 + Tds/Tis) Tiparallel = Tiseries · (1 + Tds/Tis) Tdparallel = Tiseries · Tdseries / (Tiseries + Tdseries)

IMC 튜닝 (Internal Model Control)
Morari & Zafiriou (1989)
💡 IMC 원리: 내부에 프로세스 모델을 내장한 제어 구조에서 유도된 PID 파라미터. λ (closed-loop time constant)를 유일한 튜닝 파라미터로 사용하여 설계자가 응답 속도를 직관적으로 조절 가능.
IMC-PID 튜닝 공식 (FOPDT 기반)
Kc = (τ + θ/2) / [Kp · (λ + θ/2)] Ti = τ + θ/2 Td = (τ · θ/2) / (τ + θ/2)
λ
IMC 폐루프 시정수
권장: λ ≥ θ (최소 1.0×θ 이상)
λ 증가
느리고 안정적인 응답
로버스트 특성 향상
λ 감소
빠르고 공격적인 응답
과도응답 및 오실레이션 위험
⚡ λ 선택 가이드: λ = θ (공격적), λ = 2θ (균형), λ = 4θ (보수적). 데드타임 지배 루프(r > 0.5)에서 특히 유효.
λ
Lambda (Dahlin) 튜닝
Dahlin (1968)
💡 Lambda 튜닝 원리: 폐루프 응답을 1차 지연 시스템(시정수 λ)으로 정의한 뒤 역으로 PID 파라미터를 도출. IMC와 유사하지만 미분 동작을 사용하지 않아 노이즈에 강건. 흐름량 제어(Flow Control), 압력 제어에 특히 적합.
Lambda PI 튜닝 공식 (No Derivative)
Kc = τ / [Kp · (λ + θ)] Ti = τ Td = 0   (PI only)
ℹ️ Lambda 튜닝에서 Ti = τ (프로세스 시정수)로 고정되어, 적분 시간이 프로세스 특성을 자연스럽게 반영함. Kc만 λ로 조절.
Multiple-loop Detuning Factor
λmin = 3θ   (fast / 공격적) λrec = 5θ ~ 10θ   (cascade / 다중 루프 안정화 권장)
Z
Ziegler-Nichols 스텝 응답법
ZN (1942)
⚠️ 주의: ZN은 약 25% 오버슈트(Quarter Decay Ratio)를 허용하는 공격적 설계. 정유/화학 공정의 온도, 압력, 조성 루프에는 대부분 과도하게 공격적임. 초기 튜닝값 확인 또는 교육용 목적으로 주로 사용.
ZN 스텝 응답 PID 튜닝 공식
Kc = 1.2 · τ / (Kp · θ) Ti = 2 · θ Td = 0.5 · θ
PI only
Kc = 0.9τ/(Kp·θ)
Ti = 3θ, Td = 0
P only
Kc = τ/(Kp·θ)
Ti = ∞, Td = 0
∫
ITAE 최적 튜닝
Rovira (1969)
💡 ITAE 원리: ITAE(Integral Time Absolute Error) = ∫t·|e(t)|dt 를 최소화하는 PID 파라미터를 수치적으로 도출. 초기 오차보다 장기 지속 오차에 더 큰 패널티를 부여하여 빠른 정착에 최적화.
ITAE 기반 PID 공식 — SP 추종 (Setpoint Tracking)
Kc = (0.965 / Kp) · (θ/τ)−0.85 Ti = τ / (0.796 − 0.1465 · (θ/τ)) Td = 0.308 · τ · (θ/τ)0.929
ℹ️ 유효 범위: 0.1 ≤ θ/τ ≤ 1.0. 이 범위를 벗어나면 IMC 또는 Lambda 사용 권장.
ITAE 성능 지수 정의
ITAE = ∫₀∞ t · |e(t)| dt IAE = ∫₀∞ |e(t)| dt   (단순 절대 오차 적분) ISE = ∫₀∞ e²(t) dt   (제곱 오차 적분)
C
Cohen-Coon 튜닝
Cohen & Coon (1953)
⚠️ r = θ/τ가 큰 경우(데드타임 지배 루프)에 적합하도록 설계된 고전적 방법. r > 0.3 루프에서 ZN보다 나은 성능을 제공하나, 오버슈트가 여전히 크다. 계단 교란 대응에 적합.
Cohen-Coon PID 공식 (r = θ/τ 사용)
r = θ / τ Kc = (1/Kp) · (16 + 3r) / (12r) Ti = θ · (32 + 6r) / (13 + 8r) Td = 4θ / (11 + 2r)

DCS 파라미터 단위 변환
Vendor-Specific
💡 DCS 벤더마다 PID 파라미터의 이름, 단위, 표현 방식이 상이함. 튜닝 공식 결과(Kc, Ti, Td)를 각 DCS에 입력 가능한 형태로 반드시 변환해야 함.
벤더별 파라미터 변환표
DCS 플랫폼 비례 이득 파라미터 적분 파라미터 미분 파라미터
Honeywell Experion GAIN = Kc RSET (min/rep) = Ti RATE (min) = Td
Yokogawa CENTUM VP P (%) = 100 / Kc Ti (min) = Ti Td (min) = Td
Emerson DeltaV GAIN = Kc RESET (sec/rep) = Ti×60 RATE (sec) = Td×60
ABB 800xA KP = Kc TI (sec) = Ti×60 TD (sec) = Td×60
Yokogawa 비례대(P%) ↔ Kc 변환
P (%) = 100 / Kc   (Proportional Band) Kc = 100 / P(%) 예: Kc = 2.0 → P = 50%, Kc = 0.5 → P = 200%
⚠️ Yokogawa에서 P(%)가 클수록 제어기가 더 느리고 둔감(낮은 이득). Honeywell/DeltaV의 GAIN과 반대 방향으로 작동.
Integral 단위 변환 (min ↔ sec/rep)
RESET [sec/rep] = Ti [min] × 60 RSET [min/rep] = Ti [min]   (직접 입력 — Honeywell) ※ "rep" = repeat = 반복. min/rep = 적분 동작이 비례 동작만큼 보충하는 데 걸리는 시간

폐루프 성능 지표
Performance Metrics
Rise Time (상승 시간)
PV가 SP 변화량의
10% → 90%에 도달하는 시간
단위: min
Overshoot (오버슈트)
OS (%) = (PVmax − SP) / ΔSP × 100
목표: < 5% (대부분의 공정)
Settling Time (정착 시간)
PV가 SP ± 5% (또는 ±2%)
밴드 내에 영구히 진입한 시간
단위: min
IAE (Integral Absolute Error)
IAE = ∫|SP − PV| dt
작을수록 추종 성능 우수
단위: %·min
ISE (Integral Squared Error)
ISE = ∫(SP − PV)² dt
큰 오차에 가중 패널티
단위: %²·min
Decay Ratio (감쇠비)
DR = A₂ / A₁
연속 오버슈트 피크 비율
권장: DR < 0.25 (QDR)

모델 품질 지표
Model Validation
R² — 결정계수 (Coefficient of Determination)
R² = 1 − SSres / SStot SSres = Σ(PVactual − PVmodel)² SStot = Σ(PVactual − PV̄actual)² R² = 1.0 : 완벽한 모델 | R² > 0.95 : 우수 | R² > 0.85 : 허용 | R² < 0.80 : 재식별 권장
RMSE — 평균제곱근오차 (Root Mean Squared Error)
RMSE = √[ (1/n) · Σ(PVactual − PVmodel)² ] 단위: PV와 동일. 절대 오차 크기 평가에 사용.
SNR — 신호 대 잡음비 (Signal-to-Noise Ratio)
SNR = ΔPVsignal / σnoise σ_noise: 정상 상태 PV의 표준편차 (노이즈 추정) SNR > 5 : 식별 가능 | SNR < 3 : 노이즈 과다 — 재실험 권장
〜
ACF 기반 진동 감지
Loop Oscillation
자기상관함수 (AutoCorrelation Function)
ACF(k) = Σ[(e(t) − ē) · (e(t+k) − ē)] / Σ(e(t) − ē)² e(t): 오차 신호 = SP − PV | k: lag (샘플링 수) | ē: 오차 평균
📌 진단 기준:
• ACF(k)가 특정 lag에서 강한 음(−) 피크 후 주기적 패턴 → 루프 진동
• ACF가 느리게 감쇠 (ACF(1) > 0.9) → 느린 드리프트 또는 외란
• ACF가 급격히 0으로 수렴 → 화이트 노이즈 (정상 동작)
진동 주기 추정
Tosc = 2 · kfirst_negative_peak · Δt Δt: 샘플링 간격 (min) | 첫 번째 음의 ACF 피크 위치에서 주기 추정
Oscillation Index (Harris Index 기반)
η = σ²MVC / σ²actual σ²_MVC: 최소 분산 제어(MVC) 하에서 예측 분산 | σ²_actual: 실제 PV 분산 η → 1.0 : 이상적 성능 | η < 0.5 : 성능 개선 여지 큼

Stiction 분석 — Lissajous 기법
Valve Stiction
💡 Stiction (정지마찰): 밸브 내부 패킹의 마찰로 인해 MV 신호가 변해도 밸브가 즉시 움직이지 않는 현상. 결과적으로 PV가 지속적인 한 방향 진동(square wave)을 보임.
Lissajous 플롯 (OP vs PV)
X축: OP(t) — 컨트롤러 출력 (밸브 포지션 신호) Y축: PV(t) — 프로세스 변수 Stiction 징후 → 타원이 아닌 기울어진 사각형 패턴 (parallelogram / box shape) 정상 → 타원형 (ellipse) 또는 직선 (linear)
Stiction 정량 추정 (Choudhury 방법)
S = |OPdeadband| + |OPslip| S: Stiction Band (% OP 범위) | deadband: 반응 전 불감대 | slip: 급격히 움직이는 양
📶
SNR 및 데이터 품질
Data Quality
SNR — 신호 대 잡음비
SNR [linear] = ΔPV / σnoise SNR [dB] = 20 · log₁₀(ΔPV / σnoise) ΔPV: 스텝 응답 신호 크기 (최종값 − 초기값) | σ_noise: 정상상태 구간 PV 표준편차
스텝 응답 후보 구간 점수 (Score)
Score = w₁·(ΔMV/5) + w₂·(SNR/10) + w₃·(length/τ) w₁=0.4, w₂=0.4, w₃=0.2 (기본 가중치) ΔMV ≥ 3% 이상이면 유효 스텝으로 인식

폐루프 수치 시뮬레이션
Euler Integration
FOPDT 프로세스 시뮬레이션 — Euler 1차 수치적분
// dt = 0.1 min 고정 샘플링 간격 // 데드타임 버퍼: 원형 큐(circular buffer) 사용 MVdelayed(t) = MV(t − θ)   (버퍼 크기 = round(θ/dt)) PV(t+dt) = PV(t) + dt · [ (Kp · MVdelayed(t) − PV(t)) / τ ]
PID 출력 계산 — 이산 시간 (Discrete Time)
e(t) = SP(t) − PV(t) I(t) = I(t−dt) + e(t) · dt   (적분 누적) D(t) = (e(t) − e(t−dt)) / dt   (미분 근사) MV(t) = Kc · [e(t) + I(t)/Ti + Td·D(t)] MV(t) = clamp(MV(t), 0%, 100%)   (Out-of-range 방지)
부하 외란 (Load Disturbance) 모델
PVdisturbance(t) = PV(t) + Damp · exp(−(t−td)/τd) D_amp: 외란 크기 (기본값 = 5%PV) | τ_d: 외란 시정수 (기본값 = τ/3)
✅ 시뮬레이션 시나리오:
① SP 스텝 — SP를 ΔSP(기본 10%)만큼 계단 변화
② 부하 교란 — SP 고정, 외부 교란 인가 → 회복 성능 평가
③ 복합 — SP 스텝 후 교란 동시 인가 → 최악 조건 시험
